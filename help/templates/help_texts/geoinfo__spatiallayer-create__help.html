Spatial location options
There are several ways to define a spatial location:
  • File - upload a spatial file, e.g. ESRI Shapefile, Google KMZ
  • Draw on a map - draw poins, lines and/or polygons on an interactive map
  • WKT - enter "well-know text" format string. More information @ .
  • Custom - use a custom form

General steps
  1. Add a descriptive name for the location layer you are creating. If you are adding more than one spatial location, add numbers to the layer names. Here are some examples:
    • E.g. for a development project: "DP-00XX footprint 1 WKT" and "DP-00XX footprint 2 Shapefile".
    • E.g. for a general reporting layer: "Aggregate marine bird forage areas" or "Sport fishing boat reports 2016".
  2. Select a type (then follow specific steps listed below).
  3. Adding notes is optional but strongly recommended. For example, you should add where, when and from who the data/file came from.

Instructions for using Shapefile
Follow these steps:
  1. Preparation: Shapefiles must be a zipped collection of all the individual Shapefile files. It is recommended that the name of the zip file be the same as the contained files. E.g. "project_polygon.zip" would include the following files: project_polygon.shp, and project_polygon.dbf and project_polygon.prj. Multi-part polygons are NOT allowed.
  2. (Optional) Preparation: Use ArcGIS or QGIS to examine the dbf file and determine what attribute labels are available. Make note of these labels and think about the titles you would like for individual features in the map layer.
  3. (Optional) Designate an automatically generated feature label, e.g. My layer feature %ID% %poly name%. Note that feature labels are not available from KMZ files.
  4. Select File and add (a) Title and (b) Notes as instructed above above.
  5. Click the FILE button and locate the zipped Shapefile on your local system.
  6. Click CREATE PROJECT LOCATION or CREATE LAYER. The system will parse the Shapefile, creating (1st) a layer and then (2nd) ingesting all the contained features.

Instructions for using KMZ (zipped Google KML file)
Follow these steps:
  1. Preparation: Save/export your layer(s) in Google Earth as a KMZ. Warning: don't make this too complex! Also note that feature attributes are not always ingested from a KMZ.
  2. Select File and add (a) Title and (b) Notes as instructed above above.
  3. Click the FILE button and locate the GMZ file on your local system.
  4. Click CREATE PROJECT LOCATION or CREATE LAYER. The system will parse the KMZ, creating (1st) a layer and then (2nd) ingesting all the contained features.

Instructions for using Draw on a map
Follow these steps:
  1. Select Draw on a map and add (a) Title and (b) Notes as instructed above above.
  2. Zoom and pan the map to your desired location. Use the zoom buttons, and mouse-dragging.
  3. Activate the pencil button to create one or more point features. Each single-mouse-click with the pencil button activated will add a point feature.
  4. Activate the line button to create one or more line features. Start a line with a single-mouse-click and add vertices (turns) with another single-mouse-click. End the line drawing with a double-mouse-click.
  5. Multiple line features are allowed.
  6. Activate the polygon button to create one or more polygons. Start a polygon with a single-mouse-click and add vertices (turns) with another single-mouse-click. End the polygon drawing with a double-mouse-click.
  7. It is a good habit to deactivate your tool by clicking the hand tool.
  8. Edit your features by clicking on the edit button. Point: select and move. Line: select, then move a vertice or add a vertice by dragging an intermediate vertice. Polygon: select and move any vertices, and/or create a new vertices by dragging an intermediate vertices.
  9. Delete features using the Delete all features link below the map.

Instructions for WKT
Follow these steps:
  1. Select WKT and add (a) Title and (b) Notes as instructed above above.
  2. Copy and paste the WKT text into the "Wkt" field.
  3. Click CREATE PROJECT LOCATION or CREATE LAYER. The system will parse the WKT, creating (1st) a layer and then (2nd) ingesting all the contained features.
