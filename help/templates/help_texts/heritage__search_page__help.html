How to search
Type your search string (words or phrase) into the Search field and click ENTER or the button.

What can you search?
Cedar automatically updates its search index of the following:
  • work Projects - title and fields
  • grain Collections - title and fields
  • format_list_bulleted Items - title and fields
  • folder Case Brief - title and fields
  • device_hub Synthesis - title and fields
  • flag Place - title and fields
  • layers Layer - title and fields
  • insert_drive_file Attached Files - full text, e.g. linked to a Project (manually uploaded or harvested by MailHarvest)
  • insert_drive_file Attached Files - full text, e.g. linked to an Item (manually uploaded)

What are Excerpts?
Search will show you an excerpt of the text(s) around the found search terms. Use the Show/Hide Excerpt(#) link in the Results to view the excerpt(s) for that result. You can adjust the number of characters around the search term by changing the value in 'Number of Execerpt characters' field.

Researchers may want to download the results of their search query, including Excerpts. The results are downloaded in a comma separated values (CSV) file format that can be opened and analysed in a spreadsheet program. Download results by clicking on

Advanced: Search Query Modifiers
a. Boolean modifiers

AND - AND requires both terms on either side of the AND modifier to be present for a document to match. This is the search default and is not required explicitly, but is good practice when building complex search statements: salmon AND herring
OR - OR requires that either term (or both terms) be present for a document to match: bear OR salmon
OR - NOT Requires that the following term not be present. bear NOT elk
( ) - Parentheses can be used to group search clauses into sub-queries: (bear OR salmon) AND herring ...versus ... (bear AND salmon) NOT elk
" " - Phrased searches. Enclosing search terms with quotes will look for exact phrase matches: "How to navigate in the fog with no compass"

b. Wildcard modifiers

? - ? matches a single character: bo?t will match boat and bolt.
* - * Multiple characters (matches zero or more sequential characters): fi* will match fist, first, and fish. te*t will match test and text. *est will match rest, pest, test, and smallest.

c. Combining modifiers

Example of a complex query using modifiers and wildcards: (nets OR traps) AND (sal* AND ?ear)

d. Using field names in Search
You can do very specific searches by directing search to look for value in particular fields.
Try these examples:
  • To search for Items from a particular author enter the following into the Search Field: creator:"Brown, Pat Jr. - Interviewee" or creator:Brown* (to use a wildcard too).
  • To search for Items for a spreadsheet id enter the following into the Search Field: spreadsheet_id:"220"
