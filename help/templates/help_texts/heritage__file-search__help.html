File Type Filters:
Selecting one or more document type(s) from the file type drop-down list will restrict the search results by document type. If the list is empty or if all items in the list are checked, then the search will not be restricted by file type. The Clear Filters button will reset the list of selected file types to its default setting. By default, the search does not filter results by file type.
Selecting all or none of the file types in the list are functionally equivalent
Excerpts:
Where search terms are found in a document, the text surrounding that (or those) search terms is snipped out. The size of the excerpts can be determined by adjusting the number of excerpt characters.
The default excerpt size is set to 500 characters.
Search modifiers:
OR
Requires that either term (or both terms) be present for a document to match:
bear OR salmon
AND
Requires both terms on either side of the AND modifier to be present for a document to match.

This is the search default and is not required explicitly, but is good practice when building complex search statements:

salmon AND herring
NOT
Requires that the following term not be present.
bear NOT elk
(   )
Parentheses can be used to group search clauses into sub-queries:
(bear OR salmon) AND herring
(bear AND salmon) NOT elk
"   "
Phrased searches. Enclosing search terms with quotes will look for exact phrase matches:
"How to navigate in the fog with no compass"
Wildcards:
?
Matches a single character:
bo?t will match boat and bolt.
*
Multiple characters (matches zero or more sequential characters):
fi* will match fist, first, and fish.
te*t will match test and text.
*est will match rest, pest, test, and smallest.
Example of a complex query using modifiers and wildcards:
(nets OR traps) AND (sal* AND ?ear)
